关于springmvc接口注入在多个实现类的时候，如果name和value不同报错的问题

  • Java
  • Spring

Spring注入接口的时候，如果这个接口的实现类有多个，那么name必须和value相同，否则报错；而只有一个实现类的时候则可以注入成功。

Spring 的接口注入流程

  • 声明需要注入的类,例如:@Service(value = "myService"),如果不声明value,则value默认使用类名(首字母小写);
  • 注入 有Resource和Autowired两种,例如@Resouce(name="myService"),如果没有声明name,则默认使用变量名称。

遇到这么一个问题，当Service是实现某接口的时候，例如IMyService mService，value和name又正好不一样，这个时候有时候可以成功，有时候失败。总结发现成功的时候这个接口的实现类只有一个Service，而失败的时候则是有多个实现Service。
根据spring的注入原理，spring注入一般是通过按名称和按类型注入，Resource和Autowired只是两种注入方式的优先顺序不同。

  • @Resource 默认先按照名称注入，当找不到名称一致的时候会按类型注入；
  • @Autowired 默认先按照类型注入，如果要按照名称来注入，需要结合@Qualifier使用

按照以上原理，如果一个接口有多个实现类，而且注入的时候名称还不一样，

  • 如果是@Resource注入，那么先按名称注入——失败；再按照类型注入，这个时候找到多个类，spring不知道用哪个就报错了。
  • 同理，Autowired也会报错。

所以 注入的时候如果是接口，那么要看实现类是否多个，如果多个，那么name必须和value相同，否则报错；而只有一个实现类的时候不管名称怎么样都可以注入成功。
